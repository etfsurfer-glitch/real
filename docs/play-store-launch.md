## 0. 출시 전 빌드 (님이 실행 — 키스토어 비번 필요)

버전 수정(versionName "y"→"1.0.0")이 반영됐으니 **서명된 AAB를 재빌드**합니다.

```bash
cd D:/auto/naverreal/twa
# Bubblewrap 사용 시(권장) — twa-manifest.json 기준 재생성+서명
bubblewrap build
#   → 키스토어 비밀번호 입력 프롬프트 → app-release-bundle.aab 생성(서명됨)

# 또는 gradle 직접(서명 설정이 돼 있을 때)
# ./gradlew bundleRelease
```
산출물: `app/build/outputs/bundle/release/app-release.aab` ← **이 파일을 Play Console에 업로드**

> ⚠️ versionCode는 **업로드할 때마다 +1** 해야 함(재업로드 시 1→2→3…). twa-manifest.json의 appVersionCode를 올리거나 build.gradle versionCode 증가.

## 3. Data safety (데이터 보안) 양식 답변

Play Console "데이터 보안" 설문 — 콕집 기준:
- **수집 데이터**: 이메일(로그인), 전화번호(인증·선택), 기기 식별자(푸시 알림)
- **목적**: 계정 관리, 앱 기능(알림), 본인확인
- **공유**: 제3자 공유 없음
- **암호화**: 전송 중 암호화(HTTPS) 예
- **삭제 요청**: 계정 삭제로 가능(이메일 문의)
- **위치**: 접속 IP로 지역 맞춤(정밀위치 아님) — 수집 안 함으로 처리 가능(서버 IP 기반, 저장 안 함)
